# 次世代クローズド麻雀プラットフォーム 開発指示書 (Ver.1.1)

## プロジェクトの全体コンセプト

「雀魂」レベルのエンタメ性と操作感を持つ麻雀ゲームを、完全招待制のクローズドなコミュニティで運営できるプラットフォーム。
運営側が対局の勝敗リスクを負わず、場を提供する「プラットフォーマー」として収益を上げるB2B2Cモデル。
既存の「PPPoker」のような管理構造を麻雀に最適化して実装する。

## ユーザー属性と権限構造

### システム管理者（プラットフォーマー）
- プラットフォーム全体の管理・保守。
- クラブオーナーへの「原資アイテム（ダイヤ等）」の販売。
- 全てのトランザクションと不正の監視。

### クラブオーナー
- 独自の「クラブ（箱）」を開設し、運営する。
- システムからダイヤを購入し、クラブ内通貨（ポイント）を発行する。
- エージェントの任命、およびクラブ内の全ルール設定。

### エージェント
- 自分が招待したプレイヤーを管理する。
- 担当プレイヤーへのポイント配布、および回収。
- （エージェント側のみの表示として）プレイヤーのニックネームを管理用名前に変更。

### プレイヤー
- 招待・承認されたクラブ内での対局。
- 自身の戦績やポイント残高の閲覧。

## ゲーム内通貨と経済設計

- **2段階通貨システム**
  運営が販売する「ダイヤ」を、各クラブが「ポイント」に変換して運用する。
- **ポイントの配布と回収**
  クラブオーナーまたは権限を持つエージェントのみが、プレイヤーに対してポイントを操作できる。
- **収益化ポイント**
  - 対局ごとの場代（Fee）の自動徴収。
  - ポイント送金時やクラブ維持にかかる手数料。
  - これらはシステムにより自動的に計算され、原資から差し引かれる。

## ユーザー体験（UX）と操作感

- **「雀魂」をベンチマークとしたリッチな演出**
  - 牌を捨てる際のアニメーションや音の心地よさ。
  - リーチ、ロン、ツモ時の派手なエフェクト演出。
  - 対局を盛り上げるスタンプ機能。
- **ストレスのないリアルタイム性**
  - 通信遅延（ラグ）を感じさせないスムーズな打牌。
  - アプリが落ちても即座に対局画面へ復帰できるリコネクト機能。

## 卓（ルーム）のカスタマイズ機能

オーナーは卓ごとに以下のルールを自由に設定可能。

- 3人麻雀 / 4人麻雀の切り替え。
- レート設定、および場代（Fee）の設定。
- ウマ、赤ドラ、表ドラ、後付けの有無。
- 役満の有無や特殊なローカルルールの適用。

## セキュリティと信頼性の担保

- **ブロックチェーン技術の活用**
  - 全てのポイント移動、および対局結果を不変のログとして記録する。
  - 「いつ、誰が、誰に、いくら送ったか」を誰でも事後確認でき、不正や改ざんを防ぐ。
- **不正防止フィルター**
  - GPS制限：近距離にいるユーザー同士のマッチングを制限し、コンビ打ちを防止。
  - IP制限：同一ネットワークからの複数ログインを遮断。
- **完全承認制**
  - ルームIDとパスワードに加え、オーナーの承認がない限り入室できない仕組み。

## 運用フロー

1. オーナーがクラブを開設し、エージェントやプレイヤーを招待する。
2. プレイヤーはオーナー/エージェントに連絡し、ポイントのチャージを受ける。
3. プレイヤーは保有ポイントの範囲内で、設定されたレートの卓に参加する。
4. 対局終了後、勝敗に応じたポイント移動と場代の徴収が自動で行われる。
5. エージェントは随時、傘下プレイヤーの履歴を確認し、必要に応じてポイントの回収（精算）を行う。
